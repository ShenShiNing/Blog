## 什么是 Docker？

Docker 是一个开源的容器化平台，它可以让开发者打包应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 或 Windows 操作系统中，从而实现虚拟化。Docker 容器具有以下特点：

- 轻量级：容器与主机共享操作系统，无需额外的虚拟化层
- 标准化：应用及其依赖被打包在标准化的单元中
- 可移植性：可以在任何支持 Docker 的环境中运行
- 隔离性：应用程序运行在隔离的环境中，不会相互影响

## Docker 核心概念

### 1. 镜像（Image）

镜像是一个只读的模板，包含了运行应用程序所需的所有文件和配置。特点：

- 分层构建
- 可以基于基础镜像创建
- 支持版本控制
- 可以在 Docker Hub 上共享

### 2. 容器（Container）

容器是镜像的运行实例，可以被启动、停止、删除等。特点：

- 相互隔离
- 可以持久化数据
- 可以暴露端口
- 可以设置资源限制

### 3. Dockerfile

Dockerfile 是用来构建 Docker 镜像的文本文件，包含了一系列指令。常用指令：

```dockerfile
# 基础镜像
FROM node:18-alpine

# 工作目录
WORKDIR /app

# 复制依赖文件
COPY package.json package-lock.json ./

# 安装依赖
RUN npm install

# 复制源代码
COPY . .

# 构建应用
RUN npm run build

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["npm", "start"]
```

## Docker 常用命令

### 镜像相关命令

```bash
# 构建镜像
docker build -t myapp:latest .

# 列出本地镜像
docker images

# 删除镜像
docker rmi myapp:latest

# 拉取镜像
docker pull nginx:latest
```

### 容器相关命令

```bash
# 运行容器
docker run -d -p 3000:3000 --name myapp myapp:latest

# 查看运行中的容器
docker ps

# 停止容器
docker stop myapp

# 删除容器
docker rm myapp
```

## Docker Compose

Docker Compose 用于定义和运行多容器 Docker 应用程序。示例 `docker-compose.yml`：

```yaml
version: "3"
services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
    depends_on:
      - db

  db:
    image: mongo:latest
    volumes:
      - db-data:/data/db

volumes:
  db-data:
```

## 最佳实践

1. **镜像优化**

   - 使用多阶段构建
   - 合并 RUN 命令减少层数
   - 使用 .dockerignore 排除不必要的文件

2. **安全性考虑**

   - 使用非 root 用户运行应用
   - 定期更新基础镜像
   - 扫描镜像中的安全漏洞

3. **资源管理**
   - 设置容器资源限制
   - 定期清理未使用的镜像和容器
   - 使用数据卷持久化重要数据

## 实际应用案例

以下是一个完整的 Next.js 应用部署示例：

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# 运行阶段
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/public ./public
COPY --from=builder /app/package*.json ./
RUN npm install --production
EXPOSE 3000
CMD ["npm", "start"]
```

部署命令：

```bash
# 构建镜像
docker build -t nextjs-app .

# 运行容器
docker run -d -p 3000:3000 --name nextjs-app nextjs-app
```

## 常见问题及解决方案

1. **构建失败**

   - 检查 Dockerfile 语法
   - 确保所有必要文件都被正确复制
   - 查看构建日志定位错误

2. **容器无法访问**

   - 检查端口映射配置
   - 确认防火墙设置
   - 验证应用是否正确监听端口

3. **性能问题**
   - 使用 alpine 基础镜像减小体积
   - 优化构建缓存
   - 合理设置资源限制
